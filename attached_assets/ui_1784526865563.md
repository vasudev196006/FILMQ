# Glassmorphism & Liquid Glass UI/UX Build Specification

This specification defines the complete visual recipe, physics, layout structure, component contracts, and performance rules for the unified **Glassmorphism & Liquid Glass UI System** in the **Movie Review App**.

---

## Part 1: Unified Glassmorphism Design Tokens & Uniformity Standard

To guarantee 100% visual uniformity across every component, page, and overlay, all UI elements MUST adhere to this centralized Design System.

### 1. Global Color Palette & Contrast Tokens

| Token | Light Mode Value / Class | Dark Mode Value / Class | Purpose |
|---|---|---|---|
| `bg-app` | `hsl(220, 20%, 97%)` | `hsl(225, 25%, 7%)` | Deep midnight canvas |
| `glass-surface-base` | `rgba(255, 255, 255, 0.65)` | `rgba(255, 255, 255, 0.08)` | Main panels, cards & cards |
| `glass-surface-accent` | `rgba(255, 255, 255, 0.85)` | `rgba(255, 255, 255, 0.16)` | Active pills, badges & highlights |
| `glass-border` | `rgba(255, 255, 255, 0.4)` | `rgba(255, 255, 255, 0.12)` | Subtle glass edge boundaries |
| `text-primary` | `#0F172A` (slate-900) | `#F8FAFC` (slate-50) | High hierarchy headlines & titles |
| `text-secondary` | `#475569` (slate-600) | `#94A3B8` (slate-400) | Subtitles, meta info, author text |
| `accent-glow` | `hsl(250, 84%, 67%)` (indigo) | `hsl(265, 89%, 66%)` (electric violet) | Active liquid states, ratings, CTAs |

### 2. Standardized Glass Backdrop Formulas

- **Panel & Card Glass (`.glass-panel`)**:
  ```css
  backdrop-filter: blur(16px) saturate(180%);
  -webkit-backdrop-filter: blur(16px) saturate(180%);
  background: var(--glass-surface-base);
  border: 1px solid var(--glass-border);
  box-shadow: 
    inset 1px 1px 1px 0 rgba(255, 255, 255, 0.4),
    inset -1px -1px 1px 0 rgba(255, 255, 255, 0.08),
    0 8px 32px 0 rgba(0, 0, 0, 0.12);
  ```

- **Pill & Button Glass (`.glass-pill`)**:
  ```css
  backdrop-filter: blur(12px);
  -webkit-backdrop-filter: blur(12px);
  background: var(--glass-surface-accent);
  border: 1px solid rgba(255, 255, 255, 0.25);
  box-shadow: 
    inset 1px 1px 0.5px 0 rgba(255, 255, 255, 0.5),
    inset -1px -1px 0.5px 0 rgba(0, 0, 0, 0.15),
    0 4px 12px rgba(0, 0, 0, 0.08);
  ```

### 3. Corner Radius Hierarchy

- **Large Layout Containers / Modals**: `rounded-3xl` (24px)
- **Cards & Detail Drawers**: `rounded-2xl` (16px)
- **Buttons, Badges & Filter Chips**: `rounded-full` (9999px pill) or `rounded-xl` (12px)
- **Nested Inner Highlights**: Must precisely match parent radius or use `-2px` nested offset.

### 4. Interactive Feedback & Cursor Rules

- All clickable elements MUST include `cursor-pointer`.
- Smooth transitions: `transition-all duration-300 ease-out`.
- Scale transforms on click: `active:scale-95`.
- Never shift layout on hover; use opacity, inset glow, or scale with fixed containers.


- Card is a single component, takes `movie` prop + optional callback handlers.
- If `movie.href` is set, the whole card acts as a link (`target="_blank" rel="noopener noreferrer"`).
- If not, render as a `div`.

---

### 2. Layout & Shape

- **Aspect ratio**: `2/3` (standard poster ratio) — use `aspect-[2/3]`, not fixed pixel height.
- **Corner radius**: `rounded-3xl` on outer card, matched on every inner layer.
- **Grid sizing**: Card fills its grid cell (`w-full`); grid/gallery layout is managed by parent container.
- **Poster image**: `object-cover`, `absolute inset-0`, fills card completely.

---

### 3. Card Layer Hierarchy (Z-Order)

```
<a|div card>                          z: base, rounded-3xl, overflow-hidden, relative
  <img poster>                        absolute inset-0, object-cover, transition-transform duration-500
  <gradient scrim>                    absolute inset-0, pointer-events-none, z-10
  <quick-actions overlay>             absolute top-3 right-3, z-20, flex gap-2
    <watchlist button>                glass-pill icon button
    <favorite heart button>           glass-pill icon button
  <glass info panel>                  absolute bottom-0 left-0 right-0, z-20, p-4, glass-panel
    <title + meta>                    text-white, text-white/70
    <rating pill + quick-rate trigger> glass-pill, top-right of panel
</a|div>
```

**Gradient Scrim** (sits between poster and glass panel):
```css
background: linear-gradient(
  to top,
  rgba(0,0,0,0.85) 0%,
  rgba(0,0,0,0.35) 45%,
  rgba(0,0,0,0) 70%
);
```

---

### 4. Card Motion & Parallax Easing

- **Easing**: `cubic-bezier(0.175, 0.885, 0.32, 1.2)` (slight spring overshoot).
- **Duration**: `500ms`.
- **Hover behavior**: `scale(1.03)` on card container, `scale(1.10)` on poster `<img>` for parallax depth effect.
- **GPU Promotion**: `transform: translateZ(0)`, `will-change: transform`, `backface-visibility: hidden`.

## Part 3: Fixed Scroll-Reactive Liquid Glass Search Bar Spec

The **Scroll-Reactive Liquid Glass Search Bar** is a high-performance, dynamic UI component that remains **fixed at top of viewport** while content glides underneath it. As the user scrolls up or down, the background glass lens distorts in real-time based on scroll velocity, producing organic liquid refraction over underlying content.

---

### 1. Architectural Principles & UX Behavior

1. **Fixed Floating Header Layout**:
   - Wrapper: `fixed top-6 left-1/2 -translate-x-1/2 z-50 w-full max-w-lg px-4 pointer-events-none flex justify-center`.
   - Inner interactive container: `pointer-events-auto w-full`.
   - Keeps search bar permanently visible while page content scrolls behind it.

2. **Single Global SVG Glass Filter (`<GlassFilter />`)**:
   - Rendered **ONCE** at root layout (`<GlassFilter />`).
   - Uses `<filter id="container-glass">` containing:
     - `<feTurbulence id="liquid-turbulence" type="fractalNoise" baseFrequency="0.05 0.05" numOctaves="1" seed="1" result="turbulence" />`
     - `<feGaussianBlur id="liquid-noise-blur" in="turbulence" stdDeviation="2" result="blurredNoise" />`
     - `<feDisplacementMap id="liquid-displacement" in="SourceGraphic" in2="blurredNoise" scale="70" xChannelSelector="R" yChannelSelector="B" result="displaced" />`
     - `<feGaussianBlur in="displaced" stdDeviation="4" result="finalBlur" />`
     - `<feComposite in="finalBlur" in2="finalBlur" operator="over" />`

3. **Zero-React-State Scroll Animation (`useScrollDistortion`)**:
   - Updates filter DOM attributes directly via `requestAnimationFrame` loop on window scroll.
   - Preserves 60fps / 120fps rendering by avoiding React state re-renders during scroll frames.

## Part 8: Summary Checklist for Developers

- [x] Use unified design tokens (`.glass-panel`, `.glass-pill`) across all components.
- [x] Enforce consistent corner radius scaling (`rounded-3xl` for layout, `rounded-2xl` for cards, `rounded-full` for controls).
- [x] Apply `cursor-pointer` on all interactive buttons and chips.
- [x] Integrate hover feedback without layout shifts (scale/opacity/inset glow).
- [x] Provide progressive fallback for backdrop blurs on legacy devices.
- [x] Maintain proper Z-index ordering (`z-10` scrim, `z-20` glass panels, `z-30` popovers/modals, `z-50` search bar).
